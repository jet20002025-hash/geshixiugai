# 手动触发 Vercel 部署

如果 Vercel 没有自动部署更新，可以手动触发部署：

## 方法1：通过 Vercel Dashboard

1. 登录 Vercel Dashboard：https://vercel.com/dashboard
2. 选择你的项目：`geshixiugai`（或你的项目名）
3. 进入 **Deployments** 页面
4. 点击右上角的 **"Redeploy"** 按钮
5. 选择 **"Use existing Build Cache"** 或 **"Redeploy without cache"**（建议选择后者以确保使用最新代码）
6. 点击 **"Redeploy"**
7. 等待部署完成（约 1-2 分钟）

## 方法2：通过 Git 推送触发

如果代码已经提交到 Git，但 Vercel 没有检测到：

1. 确认代码已推送到 GitHub：
   ```bash
   git status
   git log --oneline -5
   git push origin main
   ```

2. 在 Vercel Dashboard 中：
   - 进入项目 **Settings** → **Git**
   - 确认 Git 仓库连接正确
   - 检查 **Production Branch** 是否为 `main` 或 `master`

3. 如果 Git 连接有问题，可以：
   - 点击 **"Disconnect"** 然后重新连接
   - 或者手动触发部署（方法1）

## 方法3：检查 Vercel 配置

1. 进入项目 **Settings** → **General**
2. 确认 **Build Command** 和 **Output Directory** 配置正确
3. 检查 **Environment Variables** 是否都已配置

## 方法4：使用 Vercel CLI

如果安装了 Vercel CLI：

```bash
# 安装 Vercel CLI（如果未安装）
npm i -g vercel

# 登录
vercel login

# 部署
vercel --prod
```

## 检查部署状态

部署完成后，可以：

1. 在 Vercel Dashboard 的 **Deployments** 页面查看部署状态
2. 点击部署记录查看构建日志
3. 确认部署成功（状态为 "Ready"）

## 验证更新

部署完成后，访问网站并：

1. 强制刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）
2. 打开浏览器控制台（F12）
3. 查看是否有新的调试日志输出

---

**注意**：如果多次尝试后仍然没有更新，可能是：
- Git 仓库连接问题
- Vercel 项目配置问题
- 代码没有正确推送到 Git

建议先检查 Git 仓库是否有最新的提交。
