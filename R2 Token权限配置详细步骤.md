# R2 API Token 权限配置详细步骤

## 🎯 当前状态

你已经进入了自定义 Token 创建页面，现在需要配置权限。

---

## 📋 配置步骤

### 步骤 1：第一个下拉菜单（已选择）

- **当前显示**：`帐户`（Account）
- **操作**：**保持不变**，这个已经选对了 ✅

### 步骤 2：第二个下拉菜单（需要选择）

- **当前显示**：可能是 `Cloudchamber` 或其他
- **操作**：
  1. **点击第二个下拉菜单**
  2. **在下拉列表中查找**：
     - **"Cloudflare R2"** 或
     - **"R2"** 或
     - **"R2 Object Storage"**
  3. **选择 R2 相关的选项**

**如果找不到 R2 选项**：
- 尝试滚动下拉列表
- 或输入 "R2" 搜索
- 或输入 "Object Storage" 搜索

### 步骤 3：第三个下拉菜单（需要选择）

- **当前显示**：`Select...`（选择...）
- **操作**：
  1. **点击第三个下拉菜单**
  2. **选择 "Edit"**（编辑权限）
   - 不要选择 "Read"（只读），我们需要读写权限

---

## ✅ 配置完成后的样子

配置完成后，三个下拉菜单应该显示：

```
第一个：帐户 (Account)
第二个：Cloudflare R2 (或 R2)
第三个：Edit (编辑)
```

---

## 📋 继续配置

配置好权限后，继续填写其他信息：

### Token 名称

- **Token name**：`word-formatter-r2-token`

### 账号资源（Account Resources）

- **选择 "Include"**（包含）
- **在下拉菜单中选择你的账号**

### 域名资源（Zone Resources）

- **选择 "None"**（不需要）

---

## 🆘 如果找不到 R2 选项

### 可能的原因

1. **R2 服务未启用**
   - 确认你已经创建了 R2 存储桶
   - 如果还没创建，先创建存储桶

2. **权限列表需要滚动**
   - 尝试滚动下拉列表
   - 或输入关键词搜索

3. **界面版本不同**
   - 不同版本的 Cloudflare 界面可能略有不同
   - 尝试搜索 "R2" 或 "Object Storage"

### 解决方法

1. **确认 R2 已启用**：
   - 回到 R2 Dashboard，确认能看到存储桶
   - 如果能正常使用 R2，说明服务已启用

2. **尝试搜索**：
   - 在第二个下拉菜单中，尝试输入 "R2"
   - 或输入 "Object" 搜索

3. **如果还是找不到**：
   - 告诉我第二个下拉菜单中有哪些选项
   - 我会帮你找到正确的选项

## 💡 提示

- **第一个下拉菜单**：保持 "Account"（帐户）
- **第二个下拉菜单**：必须选择包含 "R2" 的选项
- **第三个下拉菜单**：必须选择 "Edit"（编辑）

2. 配置账号资源

4. 检查配置
5. 点击 "Create Token"
