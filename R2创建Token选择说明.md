# R2 创建 API Token - 选择说明

## 🎯 重要提示

**不要选择任何模板！**

页面上的模板（如"编辑区域 DNS"、"阅读账单信息"等）都不适合我们的需求。

---

## ✅ 正确操作

### 步骤 1：找到"自定义令牌"选项

1. **向下滚动页面**，找到底部的 **"自定义令牌"**（Custom Tokens）部分

2. **点击 "创建自定义令牌"** 或 **"Create Custom Token"** 按钮

   - 这个选项通常在页面最底部
   - 可能显示为按钮或链接

### 步骤 2：如果找不到"自定义令牌"选项

如果页面底部没有显示"自定义令牌"选项，可以：

1. **直接点击页面右上角或顶部的 "Create Token" 按钮**
   - 通常会直接进入自定义 Token 创建页面

2. **或者点击 "创建其他令牌" 模板**
   - 这个模板会引导你创建自定义 Token

---

## 📋 创建自定义 Token 的配置

选择"自定义令牌"后，填写以下信息：

### 基本信息
- **Token name**：`word-formatter-r2-token`

### 权限（Permissions）
1. 点击 **"Add"** 或 **"+"** 按钮
2. 选择：
   - **Account**（账号）
   - → **Cloudflare R2**（R2 存储）
   - → **Edit**（编辑权限）

### 账号资源（Account Resources）
- 选择 **"Include"**（包含）
- 在下拉菜单中选择**你的账号**

### 域名资源（Zone Resources）
- 选择 **"None"**（不需要）

### 创建并保存
1. 点击 **"Continue to summary"** 或 **"Next"**
2. 检查配置
3. 点击 **"Create Token"**
4. **立即复制保存** Access Key ID 和 Secret Access Key

---

## 🆘 如果还是找不到

### 方法 1：直接访问创建页面

尝试直接访问：
```
https://dash.cloudflare.com/profile/api-tokens/create
```

### 方法 2：使用"创建其他令牌"模板

如果页面有 **"创建其他令牌"**（Create Other Tokens）模板：
1. 点击这个模板的 **"使用模板"** 按钮
2. 然后修改权限为 R2 相关权限
