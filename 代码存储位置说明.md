## 🔄 代码更新流程

### 标准流程（推荐）

```
本地修改代码 
    ↓
提交到 GitHub（git push）
    ↓
服务器拉取更新（git pull）
    ↓
重启服务（systemctl restart）
```

### 详细步骤

#### 1. 在本地（你的电脑）修改代码

```bash
# 在本地项目目录
cd /Users/zwj/word格式修改器

# 修改代码后，提交到 GitHub
git add .
git commit -m "修改说明"
git push origin main
```

#### 2. 在服务器上更新代码

```bash
# 通过 SSH 连接到服务器
ssh root@121.199.49.1

# 进入项目目录
cd /var/www/geshixiugai

# 从 GitHub 拉取最新代码
git pull origin main

# 重启服务
sudo systemctl restart geshixiugai
```

## ⚠️ 重要说明

### ✅ 应该做的

1. **所有代码修改先提交到 GitHub**
   - 确保代码有备份
   - 方便版本管理和回滚

2. **服务器代码从 GitHub 拉取**
   - 保持代码一致性
   - 避免直接修改服务器代码

3. **敏感信息不要提交到 GitHub**
   - `.env` 文件（包含密钥）
   - 私钥和公钥文件
   - 这些文件应该在服务器上单独配置

### ❌ 不应该做的

1. **不要直接在服务器上修改代码**
   - 修改会丢失（下次 `git pull` 会被覆盖）
   - 无法版本管理

2. **不要跳过 GitHub 直接部署**
   - 失去代码备份
   - 无法追踪修改历史
